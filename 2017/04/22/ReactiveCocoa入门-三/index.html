映射

  • map：对源信号发出的数据做修改，传入一个block，类型是返回对象，参数是value，不用包装成信号，直接返回修改后的数据。
  • flattenMap：用于信号中信号，当源信号发出数据时，返回一个新的信号，但是订阅者收到的是返回信号中的值，信号可以是任意类型。

组合

  • concat（链接）
  • then（秩序）
  • merge
  • zipWith
  • combineLatest
  • reduce

concat (组合、链接)

-> 按一定顺序拼接信号，当多个信号有可能发送数据的时候，按顺序进行接收。（每个都接收）
-> 借用个形象比喻，我们可以把各个信号理解成是皇上和太子关系，皇上君临天下、唯我独尊，一人独裁，谁敢抢皇上风头。只有当上一个信号（当前皇上）sendCompleted（驾崩）了则后一个信号（当前太子）变成新的皇上，它才能掌权，做自己想干的事，而驾崩的那位谁还理你。
-> 使用场景：有两部分数据，想让上部分先执行，完成之后再执行下部分（都能获取到值）

then（秩序）

-> 按一定顺序连接信号，当前一个信号完成，才会连接后一个信号。（只接收最后一个）
-> 可以理解成派一个team去完成一项任务，每人执行其中一个步骤，环环相扣，只需要关注最后一个人去执行的最后一步是否成功，若没有走到最后一步当然也是失败了。
-> 使用场景：有两部分数据：想让上部分先进行网络请求但是过滤掉数据，然后进行下部分的，拿到下部分数据
